Area of Interest

The group.1 interested in Bovisa, Milano in the Southern-East to The Train station of Bovisa on the opposite to the Politecnico di Milano campus in Bovisa. The Area mainly consists of streets divided into 36 segments (see the Data preparation section) by the Systematica company.

Data Collection

The data were collected using the Epicollect5 APP that provides the location as well as other preferred entries that define the attributes of the specific location. In the case of the PLOS lab project, these entries were defined by the lab's tutors and thus the group was filling the entities assigned to a specified location in the road segments and upload them into the APP database.

The philosophy adopted in the data collection was:
1- The one side-walk that chooses to collect the data should fairly describe the segment parameters with no obstacles that can face the collection of the data.
2- The segments may be redivided during the collection if a significant change observed within the segments.
3- Each entry should be noted and clarify to which location and segment it belongs to ensure accurate PLOS and as a sequence Enhanced PLOS values.

The collected parameters in each segments were:
1- The length of the sidewalk (m).
2- The Width of buffer (m).
3- width of bike lane and width of the shoulder (m).
3- The number of lights.
4- The percentage of on-street parking occupancy.
5- Number of Lanes.

Data Preparation

To get the values of the different parameters to calculate the PLOS and Enhanced PLOS values, each entry should be referenced to its actual segments. Due to the relatively low accuracy of the GNSS Point Positioning of the smart-phones, the points collected data points appear with random offsets from the segments (see the picture below). So the first step of data preparation was to relocate the points to their segments manually thanks to the taken notice during data collection.

Data Processing

Once all the data aligned its segments, a buffer was performed thus the segments and their points bounded by the result buffer. Furthermore, the data and the segments were joined and as a result, the two dataset shares the same source ID.
By this step, a new problem appeared where several different segments found they have shared the same Source ID which should be unique for each segment. To fix this problem, the group while their meeting with the lab tutors and Systematica representative decided to merge these segments and all parameters values belongs to these segments were averaged.
From this step all collected data became ready for calculation and therefore, the delivered data from Systematica added to the data as attributes. Moreover, the geocoding was performed for interesting points in the area using the MMQGIS Plugin.
The PLOS values computed for each segment using the given formula and benefit from the field calculator in QGIS. From the computed PLOS and the additional parameters, the enhanced PLOS values were found. Both of PLOS and enhanced PLOS values were visualized via two different styles for each.
